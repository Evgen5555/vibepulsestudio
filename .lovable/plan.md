## Цель

Visual Edits сейчас не выбирает элементы по всему лендингу. Нужно вернуть возможность редактировать тексты и статические элементы через Visual Edits, не потеряв текущий неоновый стиль и анимации.

## План правок

1. Проверить основные секции лендинга:
   - `Hero`
   - `Nav`
   - `Services`
   - `Portfolio`
   - `Process`
   - `CtaFooter`
   - при необходимости `Marquee`

2. Найти элементы, где текст напрямую находится внутри `motion.*` компонентов.

3. Заменить такие элементы на статические HTML-теги:
   - `motion.h1` → `h1` внутри `motion.div`
   - `motion.h2` → `h2` внутри `motion.div`
   - `motion.p` → `p` внутри `motion.div`
   - `motion.span` → `span` внутри `motion.div`
   - текст внутри карточек/шагов оставить обычными `h3`, `p`, `span`

4. Анимации сохранить через внешние `motion.div` контейнеры:
   - появление секций
   - задержки
   - плавные сдвиги
   - hover/tap эффекты на кнопках и карточках

5. В `Process` оставить интерактивную анимацию таймлайна только на кругах, линии и частицах, а тексты шагов сделать максимально статичными для Visual Edits.

6. Не менять:
   - тексты
   - цвета
   - размеры
   - градиентную линию
   - пульсацию и частицы таймлайна
   - текущую адаптивность

## Проверка после реализации

После правок нужно открыть превью и проверить, что Visual Edits снова может выбирать основные текстовые элементы на странице: заголовки, описания, тексты кнопок, карточки, шаги процесса и футер.
